# 宝塔 2G 部署教程

这份教程是按你这次已经成功跑通的环境整理的最终版。

适用环境：

- 服务器：`2核 2G`
- 系统：`CentOS 7.9`
- 面板：`宝塔 Linux 面板`
- 前端：`Vue 3 + Vite`
- 后端：`Spring Boot 3.3.5`
- 数据库：`MySQL 5.7.44`
- Java：`JDK 21.0.2`

## 一、这次实际用的部署方式

这次不是用 Docker 部署的，而是用下面这套组合：

- `Nginx` 提供前端静态页面
- `Nginx` 反向代理后端接口
- `Spring Boot jar` 独立运行后端服务
- `systemd` 管理后端进程并实现开机自启
- `MySQL` 存业务数据

这是典型的：

`前后端分离 + Nginx + Java Jar + MySQL`

对于 `2G` 内存服务器，这种方式比 Docker 更省资源。

## 二、部署原理

访问链路是这样的：

1. 用户浏览器访问：

```text
http://43.242.202.68/
```

2. `Nginx` 先把前端静态文件返回给浏览器：

```text
/www/wwwroot/language-exchange-platform/frontend-dist
```

3. 浏览器里的前端页面继续请求接口：

```text
/api/*
```

4. `Nginx` 再把 `/api/` 请求转发给本机后端：

```text
http://127.0.0.1:8080
```

5. 后端 Spring Boot 再去连接 MySQL：

```text
127.0.0.1:3306
```

6. 如果有上传文件，请求 `/uploads/*` 时，也是由 Nginx 转发给后端读取本地文件。

可以把它理解成：

- `Nginx` 是门口接待
- `Spring Boot` 是业务处理员
- `MySQL` 是仓库

## 九、Nginx 站点配置

站点配置的核心内容如下：

```nginx
server {
    listen 80;
    server_name 43.242.202.68;
    index index.html index.htm default.htm default.html;
    root /www/wwwroot/language-exchange-platform/frontend-dist;

    #CERT-APPLY-CHECK--START
    include /www/server/panel/vhost/nginx/well-known/43.242.202.68.conf;
    #CERT-APPLY-CHECK--END

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8080/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /uploads/ {
        proxy_pass http://127.0.0.1:8080/uploads/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location ~ ^/(\.user.ini|\.htaccess|\.git|\.env|composer\.(json|lock)|package(-lock)?\.json|yarn\.lock|pnpm-lock\.yaml|vite\.config\.(js|ts)|Dockerfile)$ {
        return 404;
    }

    access_log  /www/wwwlogs/43.242.202.68.log;
    error_log  /www/wwwlogs/43.242.202.68.error.log;
}
```

其中最重要的 3 个点：

- `root` 指向前端打包目录
- `try_files $uri $uri/ /index.html;` 解决 Vue 路由刷新 404
- `/api/` 和 `/uploads/` 转发到后端 `8080`

## 十三、以后更新项目怎么做

如果后面你本地改了代码，更新流程就是：

1. 本地重新打包前端
2. 本地重新打包后端 jar
3. 上传新的前端 `dist`
4. 上传新的后端 `jar`
5. 重启后端服务

重启命令：

```bash
systemctl restart language-exchange
```

如果数据库结构也变了，再额外执行新的 SQL。

### 标准更新手册

这是以后最推荐的更新顺序。

#### 1. 本地重新打包前端

```bash
cd frontend
npm run build
```

打包完成后，会得到新的：

```text
frontend/dist
```

#### 2. 本地重新打包后端

```bash
cd backend
mvn clean package -DskipTests
```

打包完成后，会得到新的：

```text
backend/target/language-exchange-platform-0.0.1-SNAPSHOT.jar
```

#### 3. 上传前端文件到服务器

把本地 `frontend/dist` 里的内容覆盖到服务器目录：

```text
/www/wwwroot/language-exchange-platform/frontend-dist
```

主要会替换：

- `index.html`
- `assets/`

#### 4. 上传后端 jar 到服务器

把新的 jar 覆盖到：

```text
/www/wwwroot/language-exchange-platform/backend/language-exchange-platform-0.0.1-SNAPSHOT.jar
```

#### 5. 重启后端服务

```bash
systemctl restart language-exchange
systemctl status language-exchange
```

如果状态是：

```text
active (running)
```

说明更新成功。

#### 6. 如果更新后有异常

先看后端日志：

```bash
journalctl -u language-exchange -n 100 --no-pager
```

或者：

```bash
tail -n 100 /www/wwwroot/language-exchange-platform/backend/backend.log
```

#### 7. 什么情况下要执行 SQL

如果只是改：

- 页面样式
- 前端交互
- 后端接口逻辑
- 文案

通常不需要改数据库。

如果你改了：

- 数据表结构
- 新增字段
- 删除字段
- 索引

那就需要先执行新的 SQL，再重启后端。

#### 8. 最小更新流程

如果你以后只是普通发版，最短只要记住这三步：

1. 上传新的前端 `dist`
2. 上传新的后端 `jar`
3. 执行：

```bash
systemctl restart language-exchange
```

## 十四、安全建议

建议只对公网开放：

- `80`
- `22`
- `443` 以后配 HTTPS 时再放行

不建议长期对公网开放：

- `8080`

原因是：

- 用户访问已经走 `Nginx`
- 后端只需要给本机 Nginx 转发即可
- 关闭 `8080` 会更安全

## 十五、建议备份

至少备份这三样：

- 数据库 `language_exchange_platform`
- `/www/wwwroot/language-exchange-platform/backend/application-prod.yml`
- `/www/wwwroot/language-exchange-platform/backend/uploads`

## 十六、常见问题

### 1. 页面能打开，但登录失败

优先检查：

- 数据库是否已导入 `schema.sql`
- 数据库密码是否和 `application-prod.yml` 一致
- 后端服务是否是 `active (running)`

### 2. 页面刷新后 404

说明站点配置缺少：

```nginx
try_files $uri $uri/ /index.html;
```

### 3. 上传文件失败

检查目录是否存在：

```bash
/www/wwwroot/language-exchange-platform/backend/uploads/chat
```

### 4. 后端起不来

先看：

```bash
systemctl status language-exchange
journalctl -u language-exchange -n 100 --no-pager
```

### 5. Nginx 保存配置时报错

如果报：

```text
enable-php-00.conf not found
```

说明站点配置里错误引用了 PHP 配置文件，删除这行即可：

```nginx
include /www/server/panel/vhost/nginx/enable-php-00.conf;
```
